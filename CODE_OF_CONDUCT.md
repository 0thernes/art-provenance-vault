# Code of Conduct

Art Provenance Vault is a project about trust. That commitment to trust extends
to how we work together.

## Our Pledge

We pledge to make participation in this project a harassment-free experience
for everyone, regardless of background, experience level, or identity. We aim
to be welcoming, constructive, and honest — especially when disagreement arises
over technical or design decisions.

## Expected Behaviour

- Communicate with respect and precision. When critiquing code or design,
  address the idea, not the person.
- Give accurate, complete context when opening issues or submitting changes.
  This is a trust-system project; inaccurate reports waste everyone's time and
  can mislead.
- Acknowledge the documented boundaries of the PoC rather than treating known
  limitations as bugs or attacks.
- Credit prior work and third-party sources. The same attribution norms we
  build into the provenance ledger apply here.
- Accept and give feedback graciously.

## Unacceptable Behaviour

- Harassment, insults, or personal attacks in any communication channel.
- Misrepresenting the project's security guarantees — either to users
  (overclaiming) or in bug reports (inventing vulnerabilities).
- Submitting code that deliberately introduces security regressions or
  backdoors.
- Publishing another participant's private information without consent.
- Conduct that would be unprofessional in any collaborative technical setting.

## Enforcement

Violations can be reported to **0_0@0thernes.art** with subject line
`[APV CONDUCT]`. Reports are handled confidentially. Consequences range from a
private warning to permanent removal from the project, depending on severity
and pattern.

This code of conduct is adapted in spirit from the
[Contributor Covenant](https://www.contributor-covenant.org/), v2.1, which we
recommend reading for broader context and enforcement guidance.
